Tim Sort

Python's and Java's built-in sort — a hybrid of merge sort and insertion sort that exploits natural runs in real-world data.

Tim Sort was designed by Tim Peters in 2002 for Python and has since been adopted by Java, Android, and many other platforms. It is a hybrid algorithm that combines insertion sort for small chunks with merge sort for combining them — deliberately engineered around the structure of real-world data rather than random inputs.

Imagine you are tidying a messy bookshelf. Instead of starting from scratch, you first notice sections that are already in order ("natural runs"), sort any tiny piles quickly by hand, and then carefully interleave the ordered sections two at a time. Tim Sort does exactly this: it detects ascending and descending subsequences, reverses descending ones, extends short runs to a minimum length (minRun) via insertion sort, and then performs a series of power-of-two-aware merges.

Tim Sort's real strength is on partially sorted data — common in databases, log files, and user-edited lists — where it achieves O(n) best case because it merely detects the existing order. Its worst case matches merge sort at O(n log n) with O(n) auxiliary space.

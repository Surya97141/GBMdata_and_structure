Bucket Sort

Scatter into buckets, sort each bucket, gather — ideal for uniformly distributed data.

Bucket sort distributes elements into a number of "buckets", each covering a range of values. Each bucket is then sorted individually — typically with insertion sort or recursively with bucket sort — and finally all buckets are concatenated to produce the sorted output.

Imagine you're alphabetising a large pile of names. Instead of sorting all of them at once, you first split them into 26 smaller piles (one per letter). You then sort each small pile independently. Merging 26 almost-sorted small piles is far easier than tackling the whole heap.

When input is uniformly distributed, bucket sort achieves O(n + k) average time, where k is the number of buckets. It excels in practice for floating-point data in [0, 1). The worst case is O(n²) when all elements land in the same bucket. Extra space O(n + k) is required.

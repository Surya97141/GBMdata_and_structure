Merge Sort

Divide, sort, conquer — a classic divide-and-conquer masterpiece.

Merge sort splits the array in half repeatedly until each piece has only one element, then merges those pieces back together in sorted order. The key insight is that merging two already-sorted arrays is easy and fast — just compare the front elements and always take the smaller one.

Think of how you might merge two sorted stacks of exam papers. You look at the top sheet of each stack, take whichever name comes first alphabetically, and place it in the output pile. Repeat until one stack empties, then drop in the rest. Merge sort makes sorting a large pile trivial by recursively dividing until you have stacks of one.

This simulation uses bottom-up (iterative) merge sort for clear visual feedback: it begins by treating each element as a sorted subarray of size 1, then merges adjacent pairs to produce subarrays of size 2, then 4, and so on. Merge sort guarantees O(n log n) in all cases, making it reliable for large datasets, at the cost of O(n) auxiliary space.

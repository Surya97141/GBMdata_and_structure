Pigeonhole Sort

Sort by assigning each value its own "hole" — blazing fast when the value range equals the number of elements.

Pigeonhole Sort is a non-comparison-based integer sorting algorithm that exploits the mathematical pigeonhole principle: if you have n items and n holes, each item goes in exactly one hole. The algorithm finds the range [min, max] of the input values, creates max − min + 1 empty "holes" (buckets of capacity 1), places each element in its corresponding hole, then reads the holes back in order to reconstruct the sorted array.

Think of sorting a class of 30 students by birthday month (1–12). You create 12 slots, drop each student's name into the right slot, then read the slots in order. If every student has a unique month, each slot gets exactly one name — that is Pigeonhole Sort. Counting Sort is its generalisation that allows multiple items per hole.

The key constraint is that Pigeonhole Sort is only efficient when the range R = max − min is O(n). When R is much larger than n, the algorithm wastes time initialising and scanning mostly empty holes. For integers in a small range — scores 0–100, ASCII characters, calendar days — it achieves true O(n + R) time with zero comparisons, beating every comparison-based algorithm's Ω(n log n) lower bound.

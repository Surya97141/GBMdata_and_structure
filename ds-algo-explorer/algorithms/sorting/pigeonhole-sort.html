Pigeonhole Sort

One pigeonhole per value — place each element, then collect in order.

Pigeonhole Sort creates one "hole" (bucket that can hold multiple items) for each distinct value in the range [min, max]. Each element is placed into its corresponding hole, then holes are collected in order to reconstruct the sorted array. Unlike Counting Sort, it stores actual elements (not counts), making it stable.

Best when n ≈ range (value range close to array size). Falls apart when range ≫ n due to wasted hole-allocations.

Time
O(n + range)
Space
O(n + range)
Stable
Yes
Best for
n ≈ range

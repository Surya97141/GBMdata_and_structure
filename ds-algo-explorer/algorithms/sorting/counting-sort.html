Counting Sort

Count occurrences, then reconstruct — beats O(n log n) when the range is small.

Counting sort works by counting how many times each distinct value appears in the input. It then uses those counts to calculate each value's position in the sorted output, and fills the output array accordingly. It doesn't compare elements at all — making it a non-comparison sort.

Imagine you have a bag of coloured marbles: 3 red, 5 blue, 2 green. Instead of shuffling them into order one by one, you simply lay out exactly 3 red, then 5 blue, then 2 green. You never had to compare marbles — you just needed the counts. Counting sort works the same way with integer-valued data.

The algorithm runs in O(n + k) time where k is the range of values. When k is small relative to n, this is very fast — often faster than any comparison-based sort. However it requires O(k) extra space for the count array, and is only practical when the value range is not too large.

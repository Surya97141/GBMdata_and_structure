Strand Sort

Extract naturally sorted subsequences (strands) from the input and merge them — adaptive and excellent for nearly-sorted data.

Strand sort works by repeatedly pulling out the longest ascending subsequence starting from the beginning of the remaining input. Each extracted "strand" is already sorted. These strands are then merged into the growing output list, similar to merge sort's merge step.

The algorithm is adaptive: if the input is already nearly sorted, very few large strands are needed and it approaches O(n) performance. In the worst case (reverse-sorted input), each strand has length 1 and the algorithm degrades to O(n²). This makes it ideal for real-world datasets that often have partially-sorted structure.

Implementation-wise it's elegant: walk left-to-right keeping an element if it's greater than the last kept element, otherwise skip. After extraction, merge the strand with the output using the standard sorted-list merge.

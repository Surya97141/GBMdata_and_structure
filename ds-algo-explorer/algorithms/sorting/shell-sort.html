Shell Sort

A gap-sequence-driven insertion sort that eliminates large inversions early — dramatically faster than naive insertion sort.

Shell Sort was invented by Donald Shell in 1959 as one of the earliest improvements over insertion sort. The core idea is simple: instead of comparing adjacent elements, start by comparing elements that are far apart. This allows large values to "leap" toward their correct position in a single pass, so by the time the gap shrinks to 1 (pure insertion sort), the array is nearly sorted and almost no moves are needed.

Think of it like reorganizing a filing cabinet of 1,000 folders. Instead of moving one folder at a time from left to right, you first sort every 10th folder, then every 5th, then every 2nd, and finally adjacent pairs. Each pass leaves the collection closer to sorted order, and the last pass is nearly effortless.

The performance of Shell Sort depends critically on the gap sequence chosen. The simple halving sequence (n/2, n/4, …, 1) gives O(n²) worst case, but Knuth's sequence (1, 4, 13, 40, …) or Ciura's sequence yields near O(n^{4/3}) in practice — competitive with many O(n log n) algorithms for small-to-medium arrays.

Radix Sort

Sort digit by digit — the postal worker's algorithm.

Radix sort processes numbers digit by digit, from the least significant digit (ones) to the most significant (hundreds, thousands, etc.). For each digit position, it uses a stable sort (typically counting sort) to group elements into ten buckets (0–9) based on that digit, then reassembles them in bucket order.

Think of how a post office sorts mail: first by country, then by state, then by city, then by street. Each pass brings more order. Radix sort does the same with digits — after sorting by ones, then by tens, then by hundreds, the array ends up fully sorted.

Radix sort achieves O(d × n) time where d is the number of digits and n is the array size. For fixed-width integers this is effectively O(n). It is a non-comparison sort and scales excellently for large integer datasets. It requires O(n + k) extra space and is stable.

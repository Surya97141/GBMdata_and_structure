Cycle Sort

The theoretically optimal algorithm for minimizing write operations — each element is written at most once to its final position.

Cycle Sort is a comparison-based, in-place algorithm that achieves the minimum possible number of array writes. It decomposes a permutation into cycles, then rotates each cycle in place. For each element, it counts how many elements are smaller than it — that count is the element's correct index. It writes the element to that index, picks up the displaced element, and repeats until the cycle is closed.

Imagine distributing numbered chairs to students who are standing in random order. For each student, you count how many students should sit before them, walk that student to the correct chair, pick up the person already sitting there, and continue the chain until you return to the starting student. Each student sits down exactly once — that is exactly one write per element.

This property makes Cycle Sort ideal for scenarios where writes are extremely expensive — e.g., sorting data on flash memory (which degrades with writes), or on EEPROM chips. The trade-off is a fixed O(n²) comparison count regardless of input, since every element must be counted against all others.

Bitonic Sort

A parallel sorting network perfectly suited for GPUs — all comparisons are predetermined and data-independent.

Bitonic Sort is a comparison network-based algorithm invented by Ken Batcher in 1968. Unlike comparison-based algorithms where the next comparison depends on the result of the previous one, Bitonic Sort follows a completely fixed sequence of compare-and-swap operations — making it ideal for parallel hardware like GPUs and FPGAs where all operations in a "step" can run simultaneously.

The algorithm works in two phases: first, it constructs a bitonic sequence — one that first increases then decreases. It does this by sorting pairs of elements alternately ascending and descending. Second, it repeatedly applies a bitonic merge that splits the sequence and swaps out-of-order pairs, halving the problem each time. After log²(n) phases, the array is fully sorted.

Think of it like a telephone network where wires cross at predetermined exchange points. No matter what numbers enter, the same wires always cross — you just swap if the order is wrong. This regularity means Bitonic Sort can be hardwired in silicon. The trade-off is that it only works on arrays whose size is a power of 2, and its O(n log² n) operation count is slightly worse than O(n log n) in sequential settings.

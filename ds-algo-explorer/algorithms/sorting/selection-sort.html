Selection Sort

Repeatedly find the minimum and place it at the front — simple but always O(n²).

Selection Sort divides the array into sorted (left) and unsorted (right) halves. Each pass scans the unsorted portion to find the minimum element, then swaps it into the next sorted position. It makes at most n−1 swaps regardless of input order.

Unlike Bubble Sort there is no early-exit optimisation — the scan is always full — so performance is always Θ(n²). It is useful when write cost dominates, since writes are minimised.

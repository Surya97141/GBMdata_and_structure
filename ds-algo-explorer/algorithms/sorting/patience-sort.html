Patience Sort

Deal cards into piles like a solitaire game, then merge the piles.

Patience Sort has two phases: (1) deal each card into the leftmost pile whose top card is ≥ the card (binary search on pile tops), or start a new pile; (2) merge all piles using a min-heap. The number of piles equals the length of the longest increasing subsequence (LIS).

It is related to the patience card game and provides an elegant O(n log n) sort with an O(n log n) LIS computation as a byproduct.

Time
O(n log n)

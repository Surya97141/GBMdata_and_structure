Patience Sort

A solitaire card game turned sorting algorithm — build patience piles, then merge them for an elegant O(n log n) sort.

Patience Sort is inspired by the single-player card game Patience (also known as Klondike Solitaire). The algorithm works in two phases: first, deal cards (numbers) into piles where each card can only go on top of a pile whose top card is greater than or equal to the incoming card. If no such pile exists, start a new one. This greedy placement rule minimises the number of piles — and that minimum equals the length of the longest increasing subsequence (LIS) of the input.

Think of sorting a shuffled deck face-up: you lay each card on the leftmost pile whose top is not smaller than the new card. When finished dealing, you have several face-up stacks. The second phase performs a k-way merge across pile tops, always picking the smallest visible card — just like a dealer collecting the sorted deck from left to right.

Patience Sort is rarely used as a general-purpose sort due to its O(n) extra space for piles, but it has a beautiful connection to LIS computation and is occasionally used in practice for sorting data that arrives as a stream or is already partially ordered into increasing runs.

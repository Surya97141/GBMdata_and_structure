Heap Sort

Build a heap, extract the max — guaranteed O(n log n) in-place.

Heap sort uses a binary max-heap data structure. Phase 1 (Build Heap) rearranges the array into a max-heap where each parent node is greater than or equal to its children — this guarantees the largest element is at position 0. Phase 2 (Extract Max) repeatedly swaps the root (maximum) with the last unsorted element and then restores the heap property through a "sift-down" (heapify) operation.

Think of a sports tournament bracket where the champion always rises to the top. You first arrange all players into a valid bracket (build-heap), then each round you declare the top player the winner, remove them, and re-seed the remaining players (extract and heapify).

Heap sort is guaranteed O(n log n) in all cases, even worst case — unlike quick sort. It runs in-place with O(1) extra space. Its weakness is poor cache behaviour (it accesses memory non-sequentially), so it is typically slower in practice than quick sort despite the same asymptotic complexity. It is not stable.

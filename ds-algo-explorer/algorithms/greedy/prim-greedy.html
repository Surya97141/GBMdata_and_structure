Prim's MST Algorithm

Minimum Spanning Tree — greedily add cheapest edge connecting MST to non-MST vertex.

Prim's algorithm builds MST by growing a tree: start from any vertex, repeatedly add the minimum-weight edge that connects a vertex in the MST to one outside. The greedy choice is safe because the minimum edge crossing any cut belongs to some MST (cut property).

Green = in MST. Yellow = currently considering. Orange = candidate edge (cheapest to a non-MST vertex). Gray = not yet added.

Time (heap)
O(E log V)
Time (matrix)
O(V²)
Output
n-1 edges
vs Kruskal
Better dense graphs

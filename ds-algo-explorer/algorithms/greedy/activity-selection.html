Activity Selection

Schedule the maximum number of non-overlapping activities using a greedy finish-time strategy.

You have a list of activities, each with a start and finish time, sharing a single resource (a lecture hall, a machine). You want to select the maximum number of compatible activities — no two can overlap.

The greedy insight is non-obvious but provably optimal: always pick the activity that finishes earliest among those compatible with the last selected one. By finishing early, we leave the maximum room for future activities.

The algorithm runs in O(n log n): sort by finish time, then do a single linear scan. No DP needed — greedy gives the global optimum.

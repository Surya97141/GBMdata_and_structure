Greedy

Activity Selection Problem

Select maximum non-overlapping activities — greedy choice: earliest finish time.

Sort activities by finish time. Greedily select each activity if its start time ≥ last selected finish time. This always yields the maximum number of non-overlapping activities. The greedy choice property holds because selecting the activity that finishes earliest leaves the most room for future activities.

Green = selected. Red = skipped (overlaps). Yellow = currently considering.

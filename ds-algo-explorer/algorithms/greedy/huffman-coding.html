Huffman Coding

Build an optimal prefix-free code by greedily merging the two rarest symbols.

Huffman coding compresses data by assigning shorter bit-strings to more frequent characters. The algorithm builds a binary tree bottom-up: repeatedly extract the two nodes with the lowest frequencies from a min-heap and merge them into a parent node whose frequency is their sum.

The greedy choice — always merge the two cheapest — is provably optimal (exchange argument). The resulting tree is a prefix-free code: no codeword is a prefix of another, so decoding is unambiguous.

This is the foundation of many real compression formats (ZIP, PNG, MP3). For a string with skewed character frequencies, Huffman can shrink it to near the entropy bound.

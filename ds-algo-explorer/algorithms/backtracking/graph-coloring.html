Graph Coloring

Assign colors to graph vertices so no two adjacent vertices share the same color, using the minimum number of colors (k-coloring).

Graph coloring assigns colors to vertices such that no two connected vertices have the same color. The minimum number of colors needed is the graph's chromatic number. The famous Four Color Theorem guarantees any planar map needs at most 4 colors.

The real-world analogy is scheduling: if courses share students (edges), courses must be scheduled in different time slots (colors) to avoid conflicts. Register allocation in compilers and frequency assignment in cellular networks use the same idea.

The backtracking approach tries each color for the current node, checks all neighbors for conflicts, and recurses. If no color works it backtracks — just like trying marker colors on a map, erasing when a region's neighbors all share the color you just picked.

Time Complexity
O(k^V)
k colors, V vertices
Space Complexity
O(V)
Color assignment array
4-Color Theorem
k ≤ 4
Sufficient for any planar graph

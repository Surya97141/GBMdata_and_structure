Word Search

Search for a word in a 2D letter grid using DFS backtracking — exploring all 4 directions and retreating on mismatches.

The Word Search problem asks whether a given word can be found in a 2D grid of letters by tracing a path of adjacent cells (up, down, left, right) without reusing any cell. It is a direct application of backtracking DFS on a grid.

The algorithm starts from every cell that matches the first letter. From each starting point, it explores all four directions, marks the current cell as "in use", and recurses to match the next character. If a direction fails, it unmarks the cell and tries the next direction — classic backtracking.

This mirrors how you manually scan a word-search puzzle: you spot the first letter, trace forward in one direction, and when letters stop matching you go back to the start and try a new direction or starting cell.

Time Complexity
O(N·M·4^L)
N×M grid, word length L
Space Complexity
O(L)
Recursion depth = word length
Directions
4
Up, Down, Left, Right (no diagonals)

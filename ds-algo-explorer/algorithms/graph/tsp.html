Travelling Salesman Problem

Find the shortest tour visiting all cities exactly once — an NP-hard problem with exact DP and practical heuristics.

A travelling salesman must visit n cities and return home, covering the minimum total distance. This sounds simple but is NP-hard: there are (n−1)!/2 distinct tours, so brute force becomes infeasible around n=15. The problem models PCB drilling, delivery routing, genome sequencing, and telescope scheduling.

The Held-Karp algorithm solves TSP exactly using dynamic programming with bitmasks in O(n²·2ⁿ) — exponential but vastly better than O(n!). The 2-opt heuristic starts with any tour and repeatedly swaps two edges if doing so reduces total distance, converging to a local optimum quickly.

This simulation shows Nearest Neighbor (greedy heuristic) — always go to the nearest unvisited city — followed by optional 2-opt improvement. It's fast and typically within 20–25% of optimal.

Dijkstra's Algorithm

Greedily find the shortest path from a source to all nodes in a non-negative weighted graph.

Dijkstra's algorithm is the GPS of graph algorithms. It maintains a priority queue of nodes ordered by their current best-known distance from the source, always processing the closest unvisited node next.

Real-world analogy: spreading paint through pipes of varying widths. The paint always flows through the widest (shortest) available pipe first, guaranteeing the fastest route is found before longer ones. Works only with non-negative edge weights.

At each step, we "finalize" the node with the minimum tentative distance, then relax all its outgoing edges. A node is finalized when it is extracted from the priority queue — at that point, its distance is optimal and cannot be improved.

Time Complexity
O((V+E) log V)
Space Complexity
O(V)
Requires

Bellman-Ford Algorithm

Shortest paths even with negative edge weights — and detects negative cycles too.

Bellman-Ford extends Dijkstra's idea to handle negative edge weights. It relaxes ALL edges V-1 times. After V-1 passes, if any edge can still be relaxed, a negative-weight cycle exists — meaning shortest paths are undefined.

Real-world analogy: currency arbitrage detection. If you can exchange currencies in a cycle and end up with more money, that's a negative-weight cycle in the exchange-rate graph. Bellman-Ford flags this impossibility.

The algorithm is simpler than Dijkstra but slower at O(VE). It processes every edge in every pass, making V-1 complete sweeps. This guarantees that after pass k, all shortest paths using at most k edges are correctly computed.

O(V · E)

Kruskal's Algorithm

Sort edges by weight, greedily pick the cheapest non-cycle edge — MST in O(E log E).

Kruskal's algorithm builds a Minimum Spanning Tree by always adding the cheapest available edge that doesn't form a cycle. It starts by sorting all edges by weight, then processes them one by one, using a Union-Find data structure to efficiently detect cycles.

The Union-Find (Disjoint Set Union) structure is the key enabler: find(u) gives the component root of u, and union(u,v) merges two components. If edge (u,v) connects two different components (find(u) ≠ find(v)), add it to the MST. Otherwise skip it — adding it would create a cycle.

Kruskal's is preferred for sparse graphs where E is small relative to V². It's used in network design, clustering algorithms, and approximation algorithms for NP-hard problems. The MST weight equals the minimum cost to connect all nodes.

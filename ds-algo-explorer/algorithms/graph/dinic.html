Dinic's Algorithm

Fastest max-flow in practice — BFS level graph + DFS blocking flow, phase by phase.

Dinic's algorithm computes maximum network flow by alternating between two phases. First, BFS builds a "level graph" — a layered structure where each node's level is its BFS distance from the source. Second, DFS finds a "blocking flow" — a set of paths from source to sink through the level graph such that every path is saturated.

The genius of Dinic's is that each BFS phase strictly increases the length of augmenting paths. Since path length is bounded by V, there are at most O(V) phases. Each blocking flow phase takes O(VE) with the DFS using the "current arc" optimization. Total: O(V²E), much better than naive Ford-Fulkerson on large graphs.

Dinic's is the algorithm of choice in competitive programming and real systems like matching and scheduling. For unit-capacity graphs it runs even faster at O(E√V), making it ideal for bipartite matching.

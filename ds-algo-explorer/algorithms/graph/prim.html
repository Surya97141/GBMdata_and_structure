Prim's Algorithm

Grow the MST one vertex at a time — always pick the cheapest frontier edge.

Prim's algorithm builds the Minimum Spanning Tree by starting from a single vertex and greedily expanding. At each step, it picks the minimum-weight edge that connects a vertex already in the MST to one that isn't. Unlike Kruskal's (which sorts all edges globally), Prim's grows the tree incrementally.

The algorithm maintains a key[] array (minimum edge weight to connect each vertex to the MST) and a parent[] array (which MST vertex provides that minimum). Using a priority queue, it runs in O(E log V) time. Without a priority queue (simple array scan), it's O(V²) — better for dense graphs.

Prim's is conceptually similar to Dijkstra's shortest path algorithm. Both maintain a frontier of vertices and greedily pick the "best" next vertex. The difference: Dijkstra minimizes path length from source, while Prim minimizes edge weight to the growing tree.

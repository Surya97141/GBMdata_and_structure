Depth-First Search

Dive as deep as possible before backtracking — reveals graph structure through discovery and finish times.

DFS explores a graph by going as deep as possible along each branch before backtracking. Think of navigating a maze by always taking the first unexplored passage until hitting a dead end, then retreating to try the next option.

Real-world analogy: recursive file-system traversal. You descend into each directory fully before moving to the next sibling. DFS assigns discovery times (when a node is first entered) and finish times (when all descendants are explored).

Nodes are colored white (unvisited), gray (in stack / being processed), then black (fully explored). On a directed graph, DFS classifies edges as tree, back, forward, or cross — a key tool for detecting cycles and computing topological order.

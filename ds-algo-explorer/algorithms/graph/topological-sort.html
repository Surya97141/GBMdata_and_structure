Topological Sort

Order DAG vertices so every edge u→v has u before v — the foundation of build systems, task scheduling, and course planning.

Imagine a list of tasks where some must be completed before others: you can't compile before writing code, can't install before downloading. Topological sort produces a linear ordering of vertices in a Directed Acyclic Graph (DAG) that respects all "must come before" constraints. If there's a cycle, no such ordering exists.

Kahn's algorithm (BFS-based) maintains in-degrees for each vertex. Start with all vertices of in-degree 0 (no prerequisites), process them one by one, decrement neighbors' in-degrees, and add any that reach zero to the queue. This naturally detects cycles: if some vertices are never processed, a cycle must exist.

DFS-based topological sort completes a node only after all its descendants are finished, then prepends it to the result — naturally reversing the DFS finish order into a valid topological order.

Time
O(V + E)
Visit each vertex/edge once
Space
O(V)
Queue + in-degrees array
Requires
DAG only — cycle → no valid ordering

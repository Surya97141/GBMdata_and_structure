Monte Carlo Algorithm

Fixed runtime, probabilistic correctness — estimate π by sampling random points.

A Monte Carlo algorithm always terminates in fixed time, but may produce an incorrect (or approximate) answer with some probability. Estimating π: sample uniform random points in [-1,1]². Count those inside the unit circle (x²+y²≤1). Ratio ≈ π/4. More samples → higher accuracy. Error decreases as O(1/√n).

Green dots = inside circle. Red dots = outside circle. The estimate converges toward π as sample count grows.

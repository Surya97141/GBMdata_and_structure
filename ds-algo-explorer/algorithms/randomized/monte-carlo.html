Monte Carlo Algorithms

Always fast, probably correct — use random sampling to estimate or decide with controllable error probability.

Monte Carlo algorithms run in bounded time and always terminate, but their answers might be wrong with some small probability. The classic example: estimating π by throwing random darts at a unit square and counting how many land inside the inscribed circle. With more darts, the estimate converges to π — you control the accuracy by choosing how many samples to use.

Unlike Las Vegas algorithms (always correct), Monte Carlo accepts some probability of error in exchange for guaranteed speed. By repeating a Monte Carlo algorithm k times and taking the majority vote, the error probability drops exponentially with k — making the failure probability arbitrarily small.

Applications include numerical integration, primality testing (Miller-Rabin), rendering in computer graphics, physics simulation, option pricing in finance, and machine learning sampling methods like MCMC.

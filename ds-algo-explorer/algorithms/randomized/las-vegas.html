Las Vegas Algorithms

Always correct, randomly fast — randomness speeds up work while guaranteeing exact answers.

A Las Vegas algorithm uses randomness but always produces a correct result. The randomness affects only the running time, not the answer. Randomized QuickSort is the canonical example: pick a random pivot at each step. In the worst case it's O(n²), but in expectation it's O(n log n) — and no adversarial input can force worst-case behavior consistently.

Contrast with Monte Carlo algorithms, which can produce wrong answers but have bounded runtime. Las Vegas is "always correct, sometimes slow"; Monte Carlo is "sometimes wrong, always fast." Las Vegas is preferable when correctness is non-negotiable.

Other examples include randomized binary search trees (treaps), random hashing, and randomized median-of-medians. The key insight: randomization prevents adversarial inputs from consistently triggering worst-case behavior.

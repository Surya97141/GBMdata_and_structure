Reservoir Sampling

Sample k items uniformly at random from a stream of unknown length — using only O(k) memory.

Imagine you're reading a massive log file and need to select 5 random lines — but you don't know how long the file is, and you can only see one line at a time. Reservoir sampling (Algorithm R) solves this: fill the "reservoir" with the first k items, then for each subsequent item i, keep it with probability k/i, replacing a random existing item if kept.

The algorithm guarantees that at any point during processing, each of the items seen so far has an equal probability of being in the reservoir. It's beautifully self-correcting: the early items' probability decreases over time exactly right to maintain uniformity.

This is essential for stream processing, database sampling without replacement, A/B testing, and any application that needs to sample from big data without loading it all into memory. Vitter's Algorithm Z extends it with better performance for very large streams.

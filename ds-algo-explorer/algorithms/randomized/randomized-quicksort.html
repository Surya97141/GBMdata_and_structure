Randomized Quicksort

Classic Quicksort with a random pivot — turning worst-case O(n²) behavior from adversarial into merely astronomically unlikely.

Randomized Quicksort is identical to standard Quicksort except the pivot is chosen uniformly at random from the current subarray rather than, say, always the last element. This simple change transforms the algorithm: no adversary can consistently create worst-case O(n²) input, because the pivot choice is unpredictable. The expected number of comparisons is O(n log n) for any input.

Think of shuffling a deck before playing a card game. A cheater who knows you always cut to the bottom card can exploit you. But if you cut to a random card each hand, their strategy fails. Randomized Quicksort "shuffles" the choice of dividing point, making it immune to adversarial orderings — sorted arrays, reverse-sorted arrays, and "organ pipe" patterns all run in expected O(n log n) instead of worst-case O(n²).

The probability that any fixed element is chosen as pivot is 1/(subarray size), and the expected number of times any pair (i,j) is compared across the whole sort is O(1/|i−j|), giving the O(n log n) bound. In practice, randomized Quicksort is often faster than merge sort due to cache efficiency and lower constant factors.

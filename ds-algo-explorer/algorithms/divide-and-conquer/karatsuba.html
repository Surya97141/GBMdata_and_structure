Karatsuba Multiplication

Multiply large numbers with 3 sub-multiplications instead of 4.

Karatsuba's algorithm (1960) multiplies two n-digit numbers using only 3 recursive multiplications of n/2-digit numbers, compared to the 4 multiplications required by the naïve grade-school method. This yields complexity O(n^log₂3) ≈ O(n^1.585) vs naïve O(n²).

Split each number: x = x₁·10^(n/2) + x₀ and y = y₁·10^(n/2) + y₀. The naïve product needs x₁·y₁, x₁·y₀, x₀·y₁, x₀·y₀ — four multiplications. Karatsuba observes that the middle term x₁·y₀ + x₀·y₁ = (x₁+x₀)(y₁+y₀) − x₁·y₁ − x₀·y₀, computable from the 3 sub-products already computed — saving one multiplication at every recursion level.

The formula: xy = P1·10^n + (P2−P1−P3)·10^(n/2) + P3, where P1 = x₁·y₁, P2 = (x₁+x₀)·(y₁+y₀), P3 = x₀·y₀. For large integers (cryptography, FFT-based multiplication), Karatsuba is a significant win over schoolbook multiplication.

Time Complexity
O(n^1.585)
Sub-multiplications
3 (vs 4 naïve)
Space
O(n log n)
P1 = x₁ × y₁     P2 = (x₁+x₀) × (y₁+y₀)     P3 = x₀ × y₀
xy = P1 × 10ⁿ + (P2 − P1 − P3) × 10^(n/2) + P3

Divide & Conquer

Quick Sort

Pick a pivot, partition around it, recurse on each half.

Quick sort selects a pivot element and partitions the array into two groups: elements smaller than the pivot (left) and elements larger (right). It then recursively applies the same process to each partition. The pivot ends up in its final sorted position after each partition step.

The divide-and-conquer nature is clear: the pivot divides the problem into two independent sub-problems. Unlike merge sort, the heavy work happens during division (partitioning) rather than during combination, and no extra memory is needed — partitioning is done in-place.

Quick sort achieves O(n log n) average time with excellent cache performance, making it the fastest comparison sort in practice. The worst case O(n²) occurs when the pivot is always the smallest or largest element (already sorted input with naive pivot selection), avoided by randomized or median-of-three pivot strategies.

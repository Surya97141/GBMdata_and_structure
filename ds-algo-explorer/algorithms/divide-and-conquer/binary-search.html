Divide & Conquer

Binary Search

A classic D&C algorithm — recursively halve the problem space.

Binary search is a textbook divide-and-conquer algorithm. The D&C pattern is explicit: divide the array into two halves at the midpoint, conquer by recursing into only the relevant half, and combine trivially (no merge step needed — the result is returned directly from the subproblem).

The recursive call tree is unbalanced in general — each level only recurses into one branch. This produces a call tree of depth O(log n) with one active node per level. Each node represents the subproblem: "is the target in this subarray?" The base cases are a single element or an empty range.

Understanding binary search as D&C builds intuition for more complex divide-and-conquer algorithms. The pattern — choose a split point, determine which subproblem the answer lies in, recurse only there — appears in range trees, segment trees, and competitive programming divide-and-conquer optimizations.

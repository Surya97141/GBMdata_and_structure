Bitmask DP

Use integers as compact sets to solve subset-state DP — the classic TSP bitmask approach runs in O(n²·2ⁿ).

In many combinatorial problems, the state includes a subset of items — which cities have been visited, which tasks are assigned. Storing each subset as a bitmask (a single integer where bit i = 1 means item i is included) lets you represent all 2ⁿ subsets efficiently and look up/modify them in O(1).

The Travelling Salesman Problem (TSP) with bitmask DP: dp[mask][i] = minimum cost to visit exactly the cities in mask, ending at city i. Transitions add one more unvisited city. With n cities this is O(n²·2ⁿ) — exponential, but exact and far better than O(n!).

Bitmask tricks: mask | (1<<i) adds city i, mask & ~(1<<i) removes it, (mask >> i) & 1 checks membership. The full set is (1<<n)-1.

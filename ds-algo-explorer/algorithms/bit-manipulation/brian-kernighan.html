Brian Kernighan's Algorithm

Count set bits in O(set bits) using n & (n−1) to clear the lowest set bit each step.

Imagine counting how many lights are on in a row of switches, but you can only turn off the rightmost lit switch at each step. Brian Kernighan's algorithm does exactly that at the bit level: the expression n & (n−1) clears the lowest set bit of n in one operation, because subtracting 1 flips all bits from the lowest set bit downward.

Instead of checking every bit (which would require 32 iterations for a 32-bit integer), this algorithm only iterates once per set bit. If a number has only 3 set bits, it performs exactly 3 iterations — much faster for sparse integers.

The trick generalizes: you can use n & (n−1) to check if n is a power of 2 (result is 0), find the position of the lowest set bit, or clear it. It's a fundamental building block of efficient bit-manipulation code.

Time Complexity
O(k)
k = number of set bits
vs Naive
O(log n)
Naive checks every bit
Use Cases
Hamming weight, popcount, power-of-2 check, bit tricks

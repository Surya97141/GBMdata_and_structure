Suffix Array Construction

Build a sorted array of all suffixes in O(n log n) using the prefix-doubling (DC3/skew) approach.

A suffix array is the sorted order of all suffixes of a string. Think of it as an alphabetical index of every possible ending of a book — if the book is "banana$", the index would let you binary-search for any pattern in O(m log n) time after O(n log n) preprocessing.

The prefix-doubling algorithm (Manber & Myers) works in rounds. In round k, suffixes are ranked by their first 2^k characters. Starting from single-character ranks, each doubling step sorts by the pair (rank[i], rank[i+k]), giving new ranks. After O(log n) rounds, all ranks are unique and define the suffix array.

Combined with the LCP array (Longest Common Prefix between adjacent suffixes), the suffix array enables O(n) pattern search, substring deduplication, and efficient string comparison — the backbone of bioinformatics tools like BWA and samtools.

Z-Algorithm

Compute Z-values in O(n) — for each position, the length of the longest substring matching the string's prefix.

The Z-array for a string S has Z[i] = length of the longest substring starting at S[i] that is also a prefix of S. Z[0] is defined as 0 (or n). The algorithm maintains a "Z-box" [l, r] — the rightmost segment known to match the prefix — and uses it to skip redundant comparisons.

Think of making a rubber stamp of a word and sliding it across the text. Wherever the stamp matches the text exactly, that length is the Z-value. The clever part: if you're inside a previously computed match window, you can copy values from the mirror position instead of re-checking character by character.

Z-algorithm is the conceptual twin of KMP. To use it for pattern search: build Z on "pattern + '$' + text"; any Z[i] ≥ m indicates a match. Used in suffix array construction, string periodicity detection, and bioinformatics.

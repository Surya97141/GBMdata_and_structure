Boyer-Moore (Bad Character)

Sub-linear string search using bad character heuristic.

Boyer-Moore compares the pattern right-to-left within each window. On a mismatch, the Bad Character heuristic skips the window: if the mismatched text character appears in the pattern, shift to align it; if it doesn't appear at all, skip the entire window. In practice this is sub-linear — often O(n/m) for large alphabets.

This implementation uses the Bad Character rule only (simplified version). Pattern scans right-to-left per window; window advances left-to-right.

Best case
O(n/m)
Worst case
O(nm)

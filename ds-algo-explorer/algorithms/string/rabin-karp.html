Rabin-Karp

Rolling hash for O(n+m) average-case pattern matching.

Rabin-Karp uses a rolling hash to compare the pattern's hash against each window of the text. If hashes match, verify character-by-character (to handle collisions — "spurious hits"). The rolling hash updates in O(1) by removing the leftmost character and adding the new rightmost character.

Window (purple) = current text window being compared. Green = confirmed match. Red = spurious hit (hash matched but characters didn't). Hash = sum of char codes * base^i mod prime.

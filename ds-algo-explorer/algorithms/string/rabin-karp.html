Rabin-Karp Algorithm

Rolling-hash based pattern search: compare hash fingerprints first, then verify character-by-character only on collision.

Rabin-Karp uses a rolling hash to slide a window over the text. For each window position, it computes a hash and compares it with the pattern's hash. A full character comparison only happens when hashes match, keeping average-case performance at O(n+m).

The rolling hash magic: removing the leftmost character and adding the rightmost one takes O(1) — like a conveyor belt where you drop one item off the back and add one to the front. No recomputing the entire window from scratch.

This technique shines for multiple pattern search — store all pattern hashes in a hash set; each window's hash is checked against the set in O(1). Used in plagiarism detection, document similarity, and network packet inspection.

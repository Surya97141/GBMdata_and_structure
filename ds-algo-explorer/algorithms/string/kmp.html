KMP Algorithm

Knuth-Morris-Pratt: linear-time pattern matching by reusing previously matched characters via a failure function.

The KMP algorithm solves pattern searching in O(n+m) time — far better than the naive O(n×m). Its insight: when a mismatch occurs after several matches, the matched portion already tells us how far back to reset the pattern pointer, so the text pointer never moves backward.

Think of a word-search with a stamp. Instead of lifting the stamp and repositioning from scratch each time it doesn't fit, KMP slides the stamp smartly by consulting a pre-computed table — the failure function (also called the "partial match" table). Each entry π[i] stores the length of the longest proper prefix of pattern[0..i] that is also a suffix.

Real-world analogy: reading a library catalogue. Once you've read "encyclop" and the next letter doesn't match, you don't restart from "e"; you know "encyclop" ends with certain sub-patterns and can jump mid-way. This is used in text editors, virus scanners, and network intrusion detection.

O(m)Space (failure fn)

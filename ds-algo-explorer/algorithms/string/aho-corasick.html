Aho-Corasick Algorithm

Multi-pattern string search using a trie with failure links — find all patterns simultaneously in linear time.

Aho-Corasick is like a postal sorting machine that knows all zip codes at once. Instead of checking each pattern one by one (like KMP for each pattern separately), it builds a trie from all patterns, then adds failure links (suffix links) that let the automaton gracefully fall back to the longest proper suffix that is a prefix of some pattern.

During the search phase, the algorithm processes the text left-to-right exactly once. At each character, it either advances to a child node, follows a failure link, or uses a precomputed goto table that folds both transitions. The time complexity is O(N + M + Z) where N is text length, M is total pattern length, and Z is the number of matches.

Real-world uses include intrusion detection systems (Snort), biological sequence search (searching DNA for motifs), and search engines indexing against keyword blacklists.

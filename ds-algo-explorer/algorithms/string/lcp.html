LCP Array — Kasai's Algorithm

Compute the Longest Common Prefix array from a suffix array in O(n) time using the LCP invariant.

The LCP (Longest Common Prefix) array stores, for each position i in the suffix array, how many characters the suffix at SA[i] shares with SA[i-1]. Together with the suffix array, LCP enables counting distinct substrings, finding the longest repeated substring, and powerful range-minimum-query optimizations.

Kasai's algorithm computes LCP in O(n) using the key insight: if suffix at position i has LCP h with its predecessor in SA, then suffix at position i+1 has LCP at least h-1 with its predecessor. We process suffixes in text order (not SA order), and h never decreases by more than 1 per step, so the total work is O(n).

Analogy: imagine comparing stacks of sorted index cards. Once you know how many cards match from the top for one card, the next card in reading order can reuse almost all of that comparison work — you just drop one card from the matching stack.

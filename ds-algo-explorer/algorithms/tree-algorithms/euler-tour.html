Euler Tour

A DFS traversal that records every node entry and exit, linearizing the tree so subtree queries become array range queries.

The Euler Tour of a tree is a DFS traversal where we record the time when we enter each node (in[v]) and the time when we exit it (out[v]). These timestamps linearize the tree into an array — the key insight is that the subtree of node v corresponds exactly to the contiguous range [in[v], out[v]] in the Euler tour array.

Imagine stamping a passport when you enter and again when you leave each country. All the countries you visited "within" France (French colonies, sub-trips) have entry/exit stamps strictly between your France entry and exit stamps. A query "how many countries did I visit in France?" becomes a range count on your passport — O(1) with prefix sums, or O(log n) with a segment tree.

This flattening is powerful: subtree sum queries, subtree updates, LCA via RMQ (minimum in euler tour between two entries), and path queries on heavy-light decomposition chains all reduce to 1D range queries on the tour array — enabling highly efficient segment tree or BIT operations.

Preprocessing
O(n)
Subtree Query
O(1) to O(log n)
Tour Length
2n − 1 entries
Use Cases
Subtree sums, HLD, LCA via RMQ

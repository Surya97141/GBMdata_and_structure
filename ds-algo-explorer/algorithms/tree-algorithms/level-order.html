Level-Order Traversal

BFS on a tree — visits nodes level by level, left to right, using a queue to process them in the order they were discovered.

Level-order traversal (BFS on a tree) visits nodes level by level from top to bottom: first the root (level 0), then its children (level 1), then their children (level 2), and so on. A queue drives the process: dequeue a node, visit it, enqueue its children. The FIFO order of the queue guarantees level-by-level processing.

Think of a waterfall cascade. Water hits the root rock first, then splits to rocks on level 1, then level 2. Level-order traversal mimics this top-down spreading. It's the basis for finding the shortest path in an unweighted graph, serializing binary trees (e.g., heap array storage), and computing tree width or height level-by-level.

Unlike DFS traversals that dive deep before backtracking, BFS stays as shallow as possible. This means if you're looking for a value in a balanced tree and it's near the root, BFS finds it faster than DFS — making it the right choice for "shallowest solution" problems.

Time Complexity
O(n)
Space (queue)
O(w) — max width
Data Structure
Queue (FIFO)
Use Cases
Shortest path, Serialization, Tree width

Heavy-Light Decomposition

Partition a tree into O(log n) chains so any root-to-leaf path crosses at most O(log n) chain boundaries — enabling O(log² n) path queries.

Heavy-Light Decomposition (HLD) partitions a tree's edges into "heavy" and "light" edges. For each node, the child with the largest subtree is connected by a heavy edge; all other children get light edges. Heavy edges form contiguous chains from the root. The key theorem: any root-to-leaf path passes through at most O(log n) light edges (and thus O(log n) chain switches).

Imagine a highway system. The main highway (heavy chain) runs through the most populated corridor. Side roads (light edges) branch off into smaller regions. Any journey from city to city crosses at most O(log n) side-road junctions before reaching a highway again. HLD formalizes this intuition, guaranteeing path queries on heavy chains can be batched into a segment tree for O(log n) each — total O(log² n) per path query.

HLD is the go-to technique in competitive programming for path queries on trees: "sum/min/max of node values on the path from u to v." It reduces this to O(log n) segment tree range queries on the linearized chain array.

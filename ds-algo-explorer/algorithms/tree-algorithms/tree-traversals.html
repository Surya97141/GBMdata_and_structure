Tree Traversals

The three fundamental ways to visit every node in a binary tree — inorder, preorder, and postorder — each revealing a different structural view.

Tree traversal means visiting every node in a binary tree exactly once. The three classic DFS traversals differ only in when they visit the current node relative to its subtrees. Preorder (Root → Left → Right) visits the parent before children. Inorder (Left → Root → Right) visits nodes in sorted order for a BST. Postorder (Left → Right → Root) visits children before parents — perfect for deletion or expression evaluation.

Think of reading a book's table of contents. Preorder is like reading chapter numbers before seeing the chapter — you always know your position first. Inorder is alphabetical order in a dictionary. Postorder is like finishing all sub-tasks before marking a project complete — you "visit" (complete) children before the parent task.

All three run in O(n) time and O(h) space (where h is tree height, for the call stack). For a balanced tree h = O(log n); for a degenerate (linear) tree h = O(n).

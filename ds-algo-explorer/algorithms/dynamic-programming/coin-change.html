Coin Change

Find the minimum number of coins needed to make a target amount.

You're at a vending machine that needs exact change. Given coin denominations and a target amount, what is the fewest coins needed? Greedy fails for unusual coin sets (e.g., coins [1,3,4] and target 6 — greedy picks 4+1+1=3 coins, but 3+3=2 coins is optimal).

DP solves this cleanly: dp[w] = minimum coins to make amount w. Base case: dp[0]=0. For each amount from 1 to target, try every coin c: if c ≤ w and dp[w-c]+1 < dp[w], update dp[w]. Start with dp[w]=∞ for w>0.

After computing dp[target], backtrack through the choice array to find which coins were used. Time O(n·W), space O(W). A classic problem in competitive programming and a great interview question.

Unbounded Knapsack

Unlimited copies of each item — fill the knapsack to get maximum value.

Unlike 0/1 Knapsack, here you can take as many copies of an item as you like. Think of a candy machine: you have coins and different candy types with different weights and prices; you want maximum sweetness within your budget.

Since items can repeat, a 1-D DP suffices: dp[w] is the maximum value for weight exactly w. For each weight from 1 to W, try every item: if it fits, check if taking it (plus the best for the remaining weight) beats the current best. The key difference from 0/1 Knapsack is we look at dp[w - weight[i]] in the same row, not the previous one.

Time complexity O(n·W), space O(W). Practical in cutting-stock problems, memory allocation, and coin-change scenarios.

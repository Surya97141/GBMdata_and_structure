0/1 Knapsack

Pick items to maximise value without exceeding a weight budget — each item used at most once.

Imagine you're a thief with a backpack that can hold at most W kilograms. You see several items, each with a weight and a dollar value. You want to walk away with as much value as possible. The "0/1" means you either take an item whole or leave it — no fractions.

The DP insight: build a table where dp[i][w] is the best value achievable using the first i items with capacity w. For each item, you decide: skip it (take dp[i-1][w]) or include it (take dp[i-1][w-weight[i]] + value[i]). You pick whichever is larger.

After filling the table you backtrack from dp[n][W] to find which items were selected. Time complexity O(n·W), space O(n·W) — or O(W) with the rolling-array trick.

Edit Distance (Levenshtein)

Minimum insertions, deletions, and substitutions to transform one string to another.

Spell checkers, DNA alignment, and autocorrect all rely on edit distance. Given two strings, what is the fewest single-character edits (insert, delete, or replace) needed to turn one into the other? This is the Levenshtein distance.

DP recurrence: dp[i][j] = edit distance between the first i characters of word1 and first j characters of word2. If characters match, dp[i][j] = dp[i-1][j-1]. Otherwise, dp[i][j] = 1 + min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1]) — the three options are delete, insert, and replace respectively.

After filling the table, backtracing from dp[m][n] reconstructs the sequence of operations. Time O(m·n), space O(m·n). This same table is used for diff algorithms and bioinformatics sequence alignment.

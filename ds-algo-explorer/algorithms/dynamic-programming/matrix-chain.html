Matrix Chain Multiplication

Find the optimal parenthesisation order to minimise scalar multiplications.

Multiplying matrices is associative: (AB)C = A(BC). But the order in which you parenthesise them dramatically changes how many scalar multiplications you need. Given matrices A₁A₂…Aₙ, find the grouping that minimises total operations.

Define m[i][j] as the minimum cost to multiply matrices from i to j. For each split point k between i and j: cost = m[i][k] + m[k+1][j] + p[i-1]·p[k]·p[j], where p[] are the dimension sequence. Fill the table by chain length — length 1 costs 0; then length 2, 3, … up to n.

The algorithm runs in O(n³) and uses O(n²) space. It finds the optimal sequence for real matrix libraries (NumPy, BLAS) and is foundational in compiler optimisation for expression evaluation order.

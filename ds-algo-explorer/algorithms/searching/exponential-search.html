Exponential Search

Double the boundary until overshoot, then Binary Search the found range.

Exponential Search first finds a range [i/2, i] where the target might exist by doubling i (1, 2, 4, 8…) until a[i] ≥ target or i exceeds n. Then Binary Search is applied within that range.

Ideal for unbounded/infinite sorted arrays and when the target is near the beginning. Outperforms Binary Search when the element is near the start, as only O(log pos) work is done to reach the right range.

Time
O(log n)
Space
O(1)
Phase 1
O(log i)
find range
Phase 2
O(log i)
binary search

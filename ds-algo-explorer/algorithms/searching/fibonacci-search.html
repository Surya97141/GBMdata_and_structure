Fibonacci Search

Use Fibonacci numbers to partition the array — no division required.

Fibonacci search uses Fibonacci numbers to determine split points instead of dividing by 2. It maintains three consecutive Fibonacci numbers (fibM2, fibM1, fibM) and uses fibM2 as the offset into the array. The range shrinks by a Fibonacci step at each iteration.

The key advantage over binary search is that Fibonacci search avoids division operations, using only addition and subtraction. This was historically significant on hardware where division is expensive. The split point is always a sum of previous Fibonacci numbers, producing comparisons that are slightly biased toward the beginning of the array — useful when elements near the start are more frequently searched.

Fibonacci search has the same O(log n) asymptotic complexity as binary search, but can outperform it on hardware where addition is significantly cheaper than division. It requires the array to be sorted and of known length (or approximated).

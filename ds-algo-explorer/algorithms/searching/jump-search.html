Jump Search

Jump √n steps ahead, then do a linear scan backwards.

Jump Search works on sorted arrays by jumping ahead in fixed steps of size √n. When we overshoot (current element > target), we step back and do a linear search in the previous block. The optimal jump size √n gives O(√n) comparisons.

It sits between Linear Search O(n) and Binary Search O(log n) in complexity, but unlike Binary Search, it benefits from sequential (forward-only) access — useful on systems where backwards access is costly (magnetic tapes, block storage).

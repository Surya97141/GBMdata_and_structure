Ternary Search

Divide the search space into thirds — eliminate one-third per step.

Ternary search is a divide-and-conquer search algorithm that splits the sorted array into three parts using two midpoints, m1 and m2. By comparing the target with these two pivot values, one-third of the search space is eliminated in every iteration.

Imagine a bookshelf with books arranged by author's last name in three reading rooms. You first check the boundary between room 1 and room 2 and between room 2 and room 3. Depending on where the target author falls, you discard one entire room and continue in the remaining two-thirds — then do the same again, and so on.

Despite eliminating a third per step, ternary search is actually slightly slower than binary search in practice because it requires two comparisons per iteration vs one for binary search, and the constant factor matters. Its time complexity is O(log₃ n) = O(log n / log 3), which is asymptotically equivalent.

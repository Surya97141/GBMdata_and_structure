Linear Search

The simplest search — check every element until you find it.

Linear search, also called sequential search, scans each element of a collection one by one from the beginning until the target is found or the list ends. It places no requirement on the data — the array does not need to be sorted — making it the most general-purpose search algorithm available.

Think of it like rummaging through an unsorted drawer looking for a specific pen. You pick up each item, check it, put it aside if it is wrong, and stop the moment you find what you need. In the worst case you inspect every item; in the best case the pen is right on top.

Because of its O(n) worst-case cost, linear search is rarely the right tool for large sorted datasets, but it shines for small lists, unsorted data, or when a single pass is all that is needed (e.g., finding the maximum value).

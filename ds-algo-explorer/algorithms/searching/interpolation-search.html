Interpolation Search

Estimate where the target should be — like a smart dictionary reader.

Interpolation search improves on binary search for uniformly distributed sorted data. Instead of always probing the midpoint, it estimates the likely position of the target using a linear interpolation formula based on the values at the current bounds.

The analogy is a dictionary. If you are looking for "Zebra," you would not open to the middle — you would open near the very end, because you know Z is at the back. Interpolation search does exactly this mathematically: it computes where the target is proportionally expected to be within the current value range.

For uniformly distributed data, interpolation search achieves O(log log n) — dramatically faster than O(log n) binary search. However, for skewed distributions, it degrades to O(n) in the worst case. It works only on sorted arrays with numeric keys.

Time (Uniform)
O(log log n)
Time (Worst)
O(n)
Sorted Required
Yes (numeric)
pos = lo + floor( (target - arr[lo]) / (arr[hi] - arr[lo]) × (hi - lo) )

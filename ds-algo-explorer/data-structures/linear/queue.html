Queue

First In, First Out (FIFO) — items are served in exactly the order they arrive.

A queue is just like the checkout line at a grocery store. The first person to join the line is the first person to be served (FIFO — First In, First Out). New arrivals join at the rear; people leave from the front. Nobody skips the line.

Queues are fundamental to scheduling and buffering: the OS task scheduler queues processes waiting for the CPU, a web server queues incoming HTTP requests, and a printer spools jobs in the order they arrive. BFS (breadth-first search) on a graph uses a queue to visit nodes level by level.

Both enqueue (add to rear) and dequeue (remove from front) run in O(1) when implemented as a linked list or a circular buffer. A naive array implementation can dequeue in O(n) due to shifting, which is why real queues avoid it.

Time Complexity
O(1)
enqueue and dequeue — both constant time
Space Complexity
O(n)
One slot per element
Use Cases
BFS, task scheduling, print spooler, message buffers, rate limiting

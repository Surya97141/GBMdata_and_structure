Deque

Double-Ended Queue — O(1) insert and remove from both the front and the rear simultaneously.

A deque (pronounced "deck") is short for Double-Ended Queue. Imagine a train where carriages can be added or removed from either end of the track. Unlike a regular queue where you can only join at the rear and leave from the front, a deque opens both doors — you can enqueue or dequeue at either end in constant time.

This flexibility lets a single deque act as both a stack and a queue. It is the data structure behind Python's collections.deque, Java's ArrayDeque, and the sliding-window maximum algorithm. Browser history (back and forward navigation) also benefits from a deque's bidirectional access.

Implemented as a doubly linked list, all four operations — addFront, addRear, removeFront, removeRear — run in O(1). A circular array implementation achieves the same with better cache performance in practice.

Time Complexity
O(1)
add/remove at front and rear — all constant
Space Complexity
O(n)
One slot per element
Use Cases
Sliding window max, palindrome check, work-stealing scheduler, browser history

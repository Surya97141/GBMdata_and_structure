Stack

Last In, First Out (LIFO) — the most recent item pushed is always the first to be popped.

A stack works exactly like a stack of dinner plates. You place new plates on top (push) and always take from the top first (pop). You cannot grab a plate from the middle without removing everything above it — that is the LIFO (Last In, First Out) guarantee.

This simple rule makes stacks invaluable for tracking state that must be "unwound" in reverse order. Your browser's back-button history is a stack. Function call frames sit on the call stack. Expression parsers use stacks to match brackets and evaluate operators. Undo/redo in any editor is two stacks.

Both push and pop operate in O(1) time because they only touch the top element. A stack can be implemented as a dynamic array (push/pop at the end) or a singly linked list (push/pop at the head).

Time Complexity
O(1)
push, pop, peek — all constant time
Space Complexity
O(n)
One slot per element
Use Cases
Call stack, undo/redo, bracket matching, DFS, expression eval

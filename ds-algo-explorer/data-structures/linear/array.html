Array

A contiguous block of memory holding elements of the same type, accessed by index in O(1).

Think of an array like a row of numbered mailboxes in an apartment lobby. Each box has a fixed address (index), so you can jump directly to box #7 without opening any of the others — that is constant-time random access.

Because all elements sit side-by-side in memory, the CPU can prefetch them efficiently, making arrays cache-friendly and fast for sequential reads. The trade-off is that inserting or deleting in the middle requires shifting every element after the target position, which costs O(n) time.

Arrays underpin almost every other data structure — stacks, queues, heaps, and hash tables all commonly use arrays internally. Knowing their strengths (fast reads, cache locality) and weaknesses (fixed size, costly middle inserts) lets you choose wisely.

Time Complexity
O(1) access
O(n) insert/delete at index
Space Complexity
O(n)
One slot per element
Use Cases
Lookup tables, buffers, matrix rows, stacks, heaps, sorting algorithms

Linked List

Nodes chained by pointers — O(1) insert at head/tail, O(n) search, no wasted pre-allocated space.

Imagine a treasure hunt where each clue tells you only where the next clue is hidden. You cannot jump to clue #5 without following clues #1 through #4. That is a linked list: each node holds a value and a pointer (arrow) to the next node.

Unlike arrays, nodes do not need to be adjacent in memory. This makes insertion and deletion at the head or tail O(1) — you just rewire one or two pointers rather than shifting a whole row of elements. The price is O(n) search time and extra memory per node for the pointer.

Singly linked lists form the backbone of stacks, queues, and graph adjacency lists. Doubly linked lists power browser history and LRU caches where you need fast inserts and deletes from both ends.

Time Complexity
O(1) insert
O(n) search / delete by value
Space Complexity
O(n)
Extra pointer per node
Use Cases
Stacks, queues, adjacency lists, LRU cache, undo history

Hash Table

Key-value store with O(1) average insert, lookup, delete via hash function + chaining.

A hash table maps keys to values using a hash function h(key) = index into an array. Collisions handled by chaining (each bucket holds a linked list) or open addressing. Load factor α = n/m (n keys, m buckets) determines performance. Average O(1) search; worst case O(n) if all keys collide. Used everywhere: Python dicts, Java HashMap.

Hash Table

Key-value storage with O(1) average lookup — hash the key to a bucket, handle collisions with chaining.

How it Works

A hash table maps keys to values using a hash function. The hash function converts a key into a bucket index. When two keys land in the same bucket (a collision), we use chaining — each bucket holds a linked list of entries. Lookup: hash the key, go to that bucket, scan the chain. Average O(1), worst case O(n) if all keys collide.

Think of a library with labelled shelves: the hash function decides which shelf a book goes on. If two books share a shelf, they form a chain. Python's dict, JavaScript's Map, Java's HashMap — all hash tables under the hood.

The load factor (n/m) measures how full the table is. Above ~0.75 the table is resized to keep operations fast.

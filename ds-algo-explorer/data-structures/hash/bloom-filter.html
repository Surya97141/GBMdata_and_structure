Bloom Filter

A probabilistic membership structure — zero false negatives, tunable false positive rate, and tiny memory footprint.

How it Works

A Bloom filter uses a bit array and k hash functions. To add an element: compute all k hashes, set those bit positions to 1. To check: compute the same k positions — if all are 1, the element is possibly in the set; if any is 0, it is definitely not in the set. No false negatives. Possible false positives.

Real uses: Chrome's safe-browsing list (is this URL malicious?), Cassandra and HBase for avoiding disk lookups, Redis for caching. Advantage: a filter for 1 billion URLs might fit in just 1.2 GB at 1% false positive rate — impossible with a hash set.

Caveat: you cannot remove elements from a basic Bloom filter (removing one bit might break membership for another element that set the same bit).

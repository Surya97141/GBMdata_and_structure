Bloom Filter

Probabilistic set membership — O(1) insert/query, zero false negatives, possible false positives.

A bloom filter is a bit array of m bits with k hash functions. To add an element: set bits at h₁(x), h₂(x), ..., hₖ(x). To query: check all k positions — if any is 0, the element is definitely absent; if all are 1, the element is probably present (may be false positive). No false negatives. Space-efficient: used in databases, caches, spam filters, to avoid disk lookups for definitely-absent keys.

Purple = bit set to 1. Yellow = bit being checked. Red = false positive pattern. Bits 0 shown as empty, 1 as filled.

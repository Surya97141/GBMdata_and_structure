Hash Set

Keys only, no values. Open addressing with linear probing — blazing-fast membership tests and unique-element storage.

How it Works

A hash set stores unique keys without associated values. We use open addressing: instead of chaining, when a collision happens we probe forward to the next empty slot. The probe sequence for key k is: h(k), h(k)+1, h(k)+2 … (mod table size). Deleted cells become tombstones (marked, not truly empty) so future probes don't stop early.

Python's set uses open addressing. Adding "apple" and "apple" twice: the second add finds it already there and does nothing — guaranteeing uniqueness. Great for deduplication, membership tests (is this user banned?), and visited-node tracking in graph traversal.

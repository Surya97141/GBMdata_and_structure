Rope

Binary tree of string chunks — O(log n) insert/delete vs O(n) for plain strings.

A rope is a binary tree where each leaf holds a small string chunk and each internal node stores the total length of its left subtree (weight). Index/concatenate: traverse by weight. Split at position k: walk down splitting subtree at k. Concat: create new root. Insert/delete: split + insert/delete + concat. All O(log n). Used in text editors (Vim, Emacs ropes) for huge documents where O(n) string operations are too slow.

Each leaf (green) holds a string fragment. Internal nodes (purple) show left-subtree weight. Yellow = node being traversed during operation.

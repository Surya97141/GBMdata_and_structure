Sparse Table

O(1) Range Minimum Queries after O(n log n) preprocessing.

A Sparse Table answers Range Minimum (or Maximum) Queries in O(1) time after O(n log n) preprocessing. The key insight is that any range [l, r] of length L can be covered by two overlapping blocks of size 2^k where 2^k ≤ L. Because minimum is idempotent (min(min(A),min(A)) = min(A)), overlapping is fine.

The table st[i][j] stores the minimum over the range starting at index i with length 2^j. To query [l, r], compute k = floor(log2(r-l+1)) and return min(st[l][k], st[r-2^k+1][k]) — one comparison, no loop.

Real-world analogy: imagine a city with road-distance tables for every power-of-two stretch. Any trip can be answered by looking up just two precomputed entries that overlap in the middle.

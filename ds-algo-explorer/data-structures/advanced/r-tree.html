R-Tree

A spatial index for rectangles and regions — the data structure that powers PostGIS, spatial databases, and map tile queries worldwide.

An R-Tree (Rectangle Tree) groups nearby spatial objects (rectangles, polygons, points) into minimum bounding rectangles (MBRs). Parent nodes contain the MBR of all children below them, forming a hierarchy of nested bounding boxes. Unlike a KD-tree (which splits points), an R-tree indexes extended objects and allows overlap between sibling bounding boxes.

Imagine a library sorted by physical shelf location. Each shelf spans a floor range; each floor spans a wing; each wing spans the building. When you search for "books near coordinates (floor 3, aisle 7)", you only open the wings that overlap, then the relevant shelves — skipping most of the building. R-trees apply this same principle to maps: "find all restaurants in this viewport" skips any MBR that doesn't overlap the viewport.

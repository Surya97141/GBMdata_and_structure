Disjoint Set (Union-Find)

Near-constant time connectivity queries with path compression and union by rank.

A Disjoint Set (Union-Find) is a data structure that tracks a partition of elements into non-overlapping subsets. It supports two core operations: Union(x, y) — merge the sets containing x and y — and Find(x) — return the representative (root) of x's set.

Two key optimizations make it extraordinarily fast. Union by rank always attaches the shallower tree under the deeper one, keeping trees flat. Path compression during Find rewires every node on the search path directly to the root, so future queries are O(1) amortized. Together they achieve O(α(n)) per operation, where α is the inverse Ackermann function — effectively constant for all practical inputs.

Real-world analogy: imagine university students in study groups. When two groups merge, everyone in both groups now share one group leader. Path compression is like every student updating their records to list the top-level leader directly, skipping intermediaries.

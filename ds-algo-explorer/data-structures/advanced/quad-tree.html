Quad-Tree

A spatial tree that recursively divides 2D space into four quadrants, enabling lightning-fast region queries and collision detection.

A Quad-Tree partitions a 2D plane by recursively subdividing each region into four equal quadrants: NE, NW, SW, SE. Subdivision continues until each region contains at most one point (or a set capacity). The result is a hierarchical spatial index that mirrors a map's zoom levels — city, district, block, building.

Think of Google Maps tiles. At zoom level 1 the world splits into 4 tiles. Each tile splits into 4 more at zoom level 2, giving 16 tiles. Each zoom doubles the granularity. A quad-tree is exactly this: a hierarchical tiling adapted dynamically to where your data actually lives, so dense urban areas subdivide deeply while empty oceans stay as single large tiles.

Region queries are blazingly efficient: instead of checking every point, you only descend into quadrants that overlap your query rectangle, pruning entire branches early. This powers map rendering, collision detection in 2D games, and mesh simplification in computer graphics.

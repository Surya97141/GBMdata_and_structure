Van Emde Boas Tree

O(log log U) operations on integers by recursively dividing the universe.

A Van Emde Boas (vEB) Tree achieves O(log log U) time for insert, delete, successor, and predecessor on a universe of size U — dramatically faster than O(log n) for large U. The key idea: split the universe into √U clusters, each of size √U, and maintain a recursive vEB tree on the cluster indices (the "summary") plus one vEB tree per cluster.

For U=16 (4-bit integers), we have √16=4 clusters of size 4. An element x maps to cluster ⌊x/4⌋ at position x mod 4. Insert into both the appropriate sub-tree and the summary. Successor searches within the cluster first; if not found, uses the summary to find the next non-empty cluster, then queries its minimum. This two-level recursion drives the log log complexity.

Real-world analogy: finding the next available parking spot in a city. Instead of checking every spot, you check which block has any free spot (summary), then zoom into that block — two levels of lookup instead of exhaustive scan.

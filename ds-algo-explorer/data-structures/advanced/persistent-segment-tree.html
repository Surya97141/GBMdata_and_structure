Persistent Segment Tree

Keep all versions after updates — O(log n) per update, O(log n) per historical query.

A persistent segment tree (PST) creates a new root for each update while sharing unchanged nodes with previous versions. Each point update creates O(log n) new nodes along the updated path; all other nodes are shared. Enables querying any past version: "what was the sum of [l,r] after the k-th update?" Also used for offline k-th smallest in range via persistent merge sort tree. Space: O(n + q log n).

Cartesian Tree

A binary tree that satisfies both BST (on indices) and min-heap (on values) properties simultaneously.

A Cartesian Tree is built from a sequence of values such that it satisfies two properties at once: an in-order traversal gives the original array indices (BST property), and the root always holds the minimum value (min-heap property). This duality makes it uniquely powerful for range minimum queries and treap-based balanced BSTs.

Construction is O(n) using a stack: iterate left to right, maintaining a right-spine. For each new element, pop the stack while the top is larger, make the last popped element the left child of the new node, then push. The result encodes the entire RMQ structure implicitly — LCA of two nodes in a Cartesian Tree equals the range minimum between them.

Real-world analogy: imagine building a tournament bracket where each match is decided by the minimum — the overall minimum wins the whole bracket, and sub-brackets represent sub-ranges.

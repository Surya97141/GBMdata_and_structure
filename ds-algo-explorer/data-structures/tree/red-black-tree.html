Red-Black Tree

A self-balancing BST using node colours to enforce height bounds with fewer rotations than AVL.

Imagine a road network where every intersection is painted either red or black. The rules are: no two consecutive red lights, and every path to a dead end passes through the same number of black lights. These colour rules guarantee the tree stays roughly balanced.

A Red-Black Tree has five invariants: (1) every node is red or black, (2) the root is black, (3) every leaf (NIL) is black, (4) red nodes have only black children, and (5) all paths from a node to its descendant NILs have the same number of black nodes.

RB trees are used by Linux kernel schedulers, Java's TreeMap, and C++ STL's std::map because they offer better amortised write performance than AVL trees. They require at most two rotations per insertion and three per deletion.

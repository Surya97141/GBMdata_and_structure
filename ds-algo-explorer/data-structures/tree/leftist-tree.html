Leftist Tree

Heap-ordered binary tree with rank property — O(log n) merge. Left subtree always taller.

A leftist tree maintains two properties: (1) min-heap order (parent ≤ children); (2) leftist property (rank(left) ≥ rank(right), where rank = length of rightmost path to a leaf). Merging two leftist trees takes O(log n): merge along right spines, then swap children when leftist property is violated. Insert = merge with singleton. Extract-min = merge left and right children.

Each node labeled (value, rank). Yellow = currently merging. Orange = swap performed. Blue edge = right spine. Rank values shown in small text.

B+ Tree

All data lives in linked leaf nodes; internal nodes are purely index guides.

A B+ Tree is like a library catalogue system. The card index (internal nodes) only stores guide keys to point you to the right bookshelf section; all the actual books (data) sit on the bottom shelf (leaf nodes), and every shelf is connected to the next by a chain — making it fast to scan through ranges.

Unlike a regular B-Tree, internal nodes in a B+ Tree do not store data — only routing keys. All actual values live in leaf nodes, and leaf nodes are linked in a sorted doubly-linked list. This means range queries can be answered by finding the start leaf and then simply walking the linked list.

B+ Trees are the most common index structure in relational databases (InnoDB, PostgreSQL, SQL Server). The linked leaf list makes ORDER BY and BETWEEN queries extremely efficient.

Search
O(log n)
Navigate to leaf via internal nodes
Range Query
O(log n + k)
k = number of results

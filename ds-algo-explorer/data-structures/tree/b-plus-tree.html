B+ Tree

B-tree where all data is in leaf nodes linked in a list — optimal for range queries in databases.

A B+ tree stores all actual data in leaf nodes; internal nodes hold only routing keys. Leaf nodes are linked as a doubly-linked list, enabling efficient sequential/range scans. Internal nodes act as index — routing keys duplicate in leaves. This is the structure used by virtually all relational database indexes (MySQL InnoDB, PostgreSQL).

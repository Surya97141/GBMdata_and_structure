Binary Tree

A hierarchical structure where every node has at most two children.

A binary tree is like a company org chart — there is one boss (the root), and each person can manage at most two direct reports (left child and right child). The relationship flows strictly top-down, creating a layered hierarchy.

Every node stores a value and holds references to its left and right subtrees. Nodes with no children are called leaves. The depth of a node is how many edges separate it from the root, and the height of the tree is the maximum depth among all leaves.

Binary trees are the foundation for more specialised structures like BSTs, heaps, and expression trees. They are used in compilers to represent arithmetic expressions, in file systems for directory hierarchies, and in decision trees for machine learning.

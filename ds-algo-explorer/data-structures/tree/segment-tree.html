Segment Tree

A tree that pre-computes range aggregates for lightning-fast interval queries.

A segment tree is like a summary hierarchy for a sports league. At the bottom you have individual match scores. One level up you group matches by week and store the weekly total. Above that, monthly totals. At the top, the season total. Any span of matches can be answered by combining a few of these pre-computed summaries rather than re-reading every match.

Each internal node stores the aggregate (sum, min, or max) of a contiguous sub-array. The root covers the entire array. Queries decompose the desired range into O(log n) pre-computed segments, and point updates propagate changes upward through O(log n) ancestors.

Segment trees are used in competitive programming, game engines for spatial queries, and database query optimisers. They answer any range aggregate query in O(log n) after an O(n) build step.

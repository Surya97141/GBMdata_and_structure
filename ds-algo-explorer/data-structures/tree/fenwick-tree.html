Fenwick Tree

Also called a Binary Indexed Tree — compact prefix-sum structure using bit tricks.

Think of a team responsibility chart. Employee 1 handles only their own work. Employee 2 handles work for employees 1-2. Employee 4 handles 1-4, employee 8 handles 1-8. To find the total for positions 1-6, you only need to look at employees 6 and 4 (not all six). This is exactly how a Fenwick Tree works.

Each index i in the BIT array stores the sum for a specific range determined by the lowest set bit (LSB) of i. The range is [i - LSB(i) + 1, i]. For a prefix sum query, you jump from i to i - LSB(i) and add values. For a point update, you jump from i to i + LSB(i) and update values.

Fenwick Trees use O(n) space and support O(log n) point updates and prefix queries with extremely small constant factors. They are the go-to structure for order statistics and frequency tables in competitive programming.

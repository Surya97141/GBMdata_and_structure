B-Tree

A self-balancing search tree designed for disk storage, where nodes hold multiple keys.

A B-Tree is like a filing cabinet with labelled dividers. Each drawer (node) can hold multiple dividers (keys), and the label on each divider tells you which sub-drawer to open next. Reading a whole drawer at once is much faster than reading one sheet at a time — exactly how disk pages work.

In a B-Tree of minimum degree t, every non-root node has between t-1 and 2t-1 keys. When a node fills up (reaches 2t-1 keys), it splits: the median key moves up to the parent, and the node becomes two half-full nodes. This splitting propagates upward, potentially growing the tree height at the root.

This simulation uses order 3 (t=2, so max 3 keys per node). B-Trees power every major database engine (PostgreSQL, MySQL InnoDB, SQLite) and file systems (NTFS, HFS+, ext4).

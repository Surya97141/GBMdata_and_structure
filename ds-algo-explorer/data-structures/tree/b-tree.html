B-Tree

Self-balancing multi-way search tree — minimizes disk I/O. Used in databases and file systems.

A B-tree of order m has nodes holding up to m-1 keys and m children. All leaves at the same depth. Nodes are at least half-full. Disk block = one node → minimizes page reads. Search, insert, delete all O(log n). Used in databases (PostgreSQL btree index) and file systems (NTFS, ext4).

Yellow = node currently being searched. Purple = key found / matching node. Steps trace a search through the B-tree of order 3 (2-3 tree).

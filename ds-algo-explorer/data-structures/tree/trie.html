Trie

A prefix tree that stores strings character-by-character, enabling lightning-fast autocomplete and prefix search.

A Trie (pronounced "try", from retrieval) is a tree-shaped data structure where each node represents a single character. Words are spelled out along root-to-leaf paths, and every node at depth d stores the d-th character of a set of related strings. Nodes that mark the end of a complete word carry a special flag.

Think of your phone's contact book. When you type "Jo", it instantly narrows suggestions to "John", "Jose", "Jordan" — because every contact name was inserted into a trie, and the prefix "Jo" leads directly to only those branches. Each keystroke simply follows one edge deeper into the tree.

Tries excel when many strings share common prefixes. Lookup and insert both run in O(m) time where m is the word length — independent of how many words are stored. This makes them far faster than sorting or hashing for prefix queries.

Treap

Randomized BST — BST by key, max-heap by priority. Expected O(log n) operations.

A treap maintains two properties simultaneously: BST order by key (left key < right key) and max-heap order by random priority (parent priority > child priority). The random priorities make the tree balanced in expectation — height O(log n) with high probability. Rotations maintain both properties after insert/delete.

Each node shows (key, priority). Yellow = current node being examined. Numbers inside show key/priority. Node size proportional to priority for intuition.

Treap

A randomized BST that stays balanced on average by combining BST key ordering with random heap priorities — no complex rebalancing needed.

A Treap (Tree + Heap) is a binary tree where each node carries two values: a key (following BST order — left keys smaller, right keys larger) and a priority (random number, following max-heap order — parent priority higher than children). Because priorities are random, the resulting tree shape has the same distribution as a random BST, which has expected height O(log n).

Think of a lottery-controlled filing system. Documents (keys) are filed in alphabetical order for fast lookup, but each document also has a randomly drawn lottery ticket (priority). Tickets determine the vertical arrangement — higher ticket always sits above lower tickets. This lottery ensures no one can consistently force a worst-case arrangement.

Insertions do a BST insert, then use rotations to restore the heap priority property. Deletions assign the node priority -∞, sift it down to a leaf using rotations, then remove it. The randomness eliminates the need for explicit balance tracking, making the code simpler than AVL or red-black trees.
